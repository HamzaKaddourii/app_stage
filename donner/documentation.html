Système de Gestion des Logements InfoDesign

Documentation complète et cahier des charges pour rapport de stage

1. Présentation du Projet

Le Système de Gestion des Logements InfoDesign est une application web moderne développée pour gérer les réservations de salles de l'entreprise InfoDesign. Cette plateforme permet aux utilisateurs de visualiser les salles disponibles, effectuer des réservations, gérer leurs bons d'achat et soumettre des demandes personnalisées. L'application dispose également d'un tableau de bord administrateur pour superviser l'ensemble du système.

1.1 Contexte du Projet

InfoDesign, entreprise spécialisée dans la formation et le développement informatique, dispose de plusieurs salles qu'elle loue pour des événements professionnels. Pour optimiser la gestion de ces espaces et faciliter le processus de réservation, l'entreprise a souhaité mettre en place une plateforme web dédiée.

1.2 Objectifs du Projet

  • Faciliter la visualisation et la réservation des salles disponibles
  • Automatiser la gestion des réservations et réduire les erreurs administratives
  • Mettre en place un système de fidélisation avec des bons d'achat
  • Permettre aux utilisateurs de soumettre des demandes spécifiques
  • Offrir aux administrateurs une vision globale et des outils de gestion efficaces

2. Cahier des Charges

2.1 Exigences Fonctionnelles

Gestion des Utilisateurs

  • Inscription et connexion des utilisateurs
  • Authentification sécurisée avec tokens
  • Profils utilisateurs modifiables
  • Différenciation des rôles (utilisateur/administrateur)
  • Récupération de mot de passe par email

Gestion des Salles

  • Catalogue de salles avec détails (capacité, équipements, prix)
  • Filtrage et recherche de salles
  • Visualisation des disponibilités
  • Galerie d'images pour chaque salle

Système de Réservation

  • Sélection des dates et heures
  • Calcul automatique des coûts
  • Vérification des disponibilités en temps réel
  • Historique des réservations pour l'utilisateur

Gestion des Bons d'Achat

  • Génération automatique après validation de réservation
  • Valeur de 10% du montant de la réservation
  • Liste des bons disponibles pour l'utilisateur
  • Génération de PDF pour chaque bon
  • Système de codes uniques

Demandes Personnalisées

  • Formulaire de demande spécifique
  • Suivi de l'état des demandes
  • Notification des réponses

Administration

  • Tableau de bord avec statistiques
  • Gestion des salles (ajout, modification, suppression)
  • Validation des réservations
  • Traitement des demandes personnalisées
  • Supervision des utilisateurs

2.2 Exigences Non Fonctionnelles

  • Performance : Temps de réponse rapide, chargement optimisé des pages
  • Sécurité : Protection des données utilisateurs, authentification sécurisée, validation des entrées
  • Disponibilité : Application accessible 24/7
  • Compatibilité : Interface responsive pour ordinateurs, tablettes et smartphones
  • Maintenabilité : Structure modulaire du code, documentation
  • Évolutivité : Architecture permettant l'ajout futur de fonctionnalités

3.2 Structure de la Base de Données

L'application utilise une base de données relationnelle MySQL avec les tables principales suivantes :

Table Description Relations
users Stocke les informations des utilisateurs 1:N avec reservations, bons_achat, demandes_personnalisees
salles Contient les informations des salles disponibles 1:N avec reservations
reservations Enregistre les réservations effectuées N:1 avec users, salles; 1:1 avec bons_achat
bons_achat Stocke les bons d'achat générés N:1 avec users; 1:1 avec reservations
demandes_personnalisees Contient les demandes spécifiques des utilisateurs N:1 avec users

3.3 Architecture de l'Application

L'application suit une architecture client-serveur avec une séparation claire entre le frontend et le backend :

  • Frontend (React) : Interface utilisateur, gestion d'état, communication avec l'API
  • Backend (Laravel) : API RESTful, logique métier, accès à la base de données, authentification

Les deux parties communiquent via des requêtes HTTP en format JSON.

5. Installation et Configuration du Projet

5.1 Prérequis

  • PHP 8.0 ou supérieur
  • Composer
  • Node.js et npm
  • MySQL
  • Git

5.2 Installation du Backend (Laravel)

git clone [repository-url]
cd gestion-logement
composer install
cp .env.example .env
php artisan key:generate
# Configurer la base de données dans .env
php artisan migrate --seed
php artisan serve

5.3 Installation du Frontend (React)

cd front-end-logement
npm install
# Configurer l'URL de l'API si nécessaire
npm start
